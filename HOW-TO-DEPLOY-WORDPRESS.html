How to Replace Your WordPress Site

Step-by-step guide for GoDaddy hosting — written for beginners

⏱ 45–60 minutes total 🟢 Beginner-friendly 🔒 Your old site is safely backed up
🛡️
Your old WordPress site is NOT deleted. Step 1 saves a full backup. You can restore it any time — it's your safety net before anything else changes.

The Plan — Plain English

Your current site runs on WordPress (a database + PHP files). Your new site is plain HTML — pre-built, faster, and simpler. Here's the order of operations:

  • 1 Back up your old WordPress site (safety net)
  • 2 Connect GoHighLevel so your contact form sends leads
  • 3 Zip up the new site folder
  • 4 Log into GoDaddy and open the File Manager
  • 5 Delete the old WordPress files from the server
  • 6 Upload and unzip the new site
  • 7 Move files into the right place
  • 8 Create the .htaccess server rules file
  • 9 Test everything works
  • 10 Submit your sitemap to Google
1
Back Up Your WordPress Site
⏱ About 10 minutes — do NOT skip this

Option A — From Inside WordPress (Easiest)

  1. Open your browser → go to disaster911.net/wp-admin → log in
  2. On the left menu, click Plugins → Add New
  3. Search for UpdraftPlus → click Install Now → Activate
  4. Go to Settings → UpdraftPlus Backups
  5. Click the big Backup Now button
  6. Check both boxes: Include your database and Include your files → click Backup Now
  7. Wait for green checkmarks to appear (5–10 minutes)
  8. Click the Existing Backups tab → click Download next to each piece (database, plugins, themes, uploads)
  9. Save all downloaded files into a new folder on your Desktop called WP-Backup-OLD

Option B — From GoDaddy cPanel (Alternative)

  1. Log into godaddy.com → My Products → Web Hosting → Manage → cPanel
  2. Scroll to the Files section → click Backup
  3. Click Download a MySQL Database Backup → select your database → download it
  4. Click Download a Home Directory Backup → save that file too
  5. Save everything into a folder on your Desktop called WP-Backup-OLD
2
Connect GoHighLevel to Your Contact Form
⏱ 5 minutes — do this BEFORE uploading anything
ℹ️
Without this step, your contact form shows a "success" message but no lead goes into your GHL inbox. Do this first while the files are still easy to find on your Desktop.

Find Your GHL Webhook URL

  1. Log into GoHighLevel at app.gohighlevel.com
  2. Click the Settings gear icon (bottom left)
  3. Click Integrations → scroll down → click Webhooks
  4. Click + Add Webhook
  5. Name it: Website Contact Form
  6. For the trigger/event, choose: Contact Created
  7. Copy the URL it gives you — paste it into a Notepad file temporarily
⚠️
Can't find Webhooks in GHL? Use a GHL embedded form instead: GHL → Sites → Forms → pick your form → Embed → copy the iframe code. You can paste it into the contact page later.

Paste the Webhook URL Into the Site File

  1. On your Desktop, open the DisasterResponse-Build-2026-02-27 folder
  2. Find the file called script.js → right-click it → Open with → Notepad
  3. Near the very top of the file, you'll see this line:
    // Paste your GoHighLevel webhook URL between the quotes below const GHL_WEBHOOK_URL = 'YOUR_GHL_WEBHOOK_URL_HERE';
  4. Replace YOUR_GHL_WEBHOOK_URL_HERE with the URL you copied from GHL. It should look like this when done:
    const GHL_WEBHOOK_URL = 'https://services.leadconnectorhq.com/hooks/abc123/webhook-trigger/xyz';
  5. Press Ctrl + S to save → close Notepad
3
Zip Your New Site Folder
⏱ 1–2 minutes
  1. Go to your Desktop → find the folder: DisasterResponse-Build-2026-02-27
  2. Right-click the folder
  3. Click Compress to ZIP file (Windows 11) — or — Send to → Compressed (zipped) folder (older Windows)
  4. A new file appears: DisasterResponse-Build-2026-02-27.zip
ℹ️
The zip takes 1–2 minutes because the site has 230+ pages. That's normal — let it finish.
4
Log Into GoDaddy & Open File Manager
⏱ 3 minutes
  1. Go to godaddy.com → log in with your GoDaddy account
  2. Click your name (top right) → My Products
  3. Find Web Hosting in the list → click Manage
  4. Click the cPanel button (a new tab opens — this is your server's control panel)
  5. In cPanel, find the Files section → click File Manager
  6. A window opens that works like Windows Explorer, but for your server
5
Remove the Old WordPress Files
⏱ About 10 minutes
⚠️
This removes the WordPress files only — NOT the database. Your database (old posts and content) stays safely on GoDaddy's servers unless you specifically delete it, which we're not doing.

Navigate to Your Site Files

  1. In File Manager, click the folder called public_html on the left side. Everything inside here is what visitors see when they go to your website.

Select & Delete These Folders

📁 wp-admin 📁 wp-includes 📁 wp-content

Select & Delete These Files

index.php wp-config.php wp-login.php wp-cron.php wp-settings.php wp-blog-header.php wp-activate.php wp-comments-post.php wp-links-opml.php wp-load.php wp-mail.php wp-signup.php wp-trackback.php xmlrpc.php .htaccess

How to Delete Them

  1. Click the first file/folder to select it
  2. Hold Ctrl and click each additional item to add it to the selection
  3. Click Delete in the top toolbar
  4. Check the box that says Skip the trash (permanent delete)
  5. Click Confirm
✅
Can't find all of them? That's fine. Some may not exist on your server. Just delete what you can see and move on.
6
Upload the New Site
⏱ About 15 minutes (large upload)
  1. Make sure you're still inside public_html in File Manager
  2. Click Upload in the top toolbar — a new upload tab opens
  3. Click Select File (or drag the zip into the upload area)
  4. Navigate to your Desktop → select DisasterResponse-Build-2026-02-27.zip
  5. Watch the progress bar — wait for it to reach 100% before doing anything else
  6. Close the upload tab when done
  7. Go back to the main File Manager tab and press F5 to refresh
7
Extract & Move Files Into Place
⏱ About 10 minutes

Extract (Unzip) the New Site

  1. In File Manager inside public_html, click once on DisasterResponse-Build-2026-02-27.zip to highlight it
  2. Click Extract in the top toolbar
  3. A box asks where to extract — make sure it says /public_html (type it in if not)
  4. Click Extract Files → wait 2–3 minutes for it to finish

Move Files to the Right Place

After extracting, the files land in a sub-folder: public_html/DisasterResponse-Build-2026-02-27/. They need to be directly inside public_html/.

  1. Double-click the DisasterResponse-Build-2026-02-27 sub-folder to open it
  2. Press Ctrl + A to select everything inside
  3. Click Move in the toolbar
  4. In the box that appears, type: /public_html
  5. Click Move Files
  6. If it asks about overwriting existing files — click Yes / Overwrite
✅
Go back to public_html (click it on the left side). If you see index.html sitting there directly, it worked.

Clean Up

  1. Click the now-empty DisasterResponse-Build-2026-02-27 folder → Delete
  2. Click DisasterResponse-Build-2026-02-27.zip → Delete
8
Create the .htaccess File
⏱ 5 minutes — required for city pages to work
⚠️
Without this file, clicking any city page link will give a "404 Not Found" error. This file is the server's traffic rulebook — it must be there.
  1. Make sure you're in public_html in File Manager
  2. Click + File in the top toolbar
  3. Type the filename exactly: .htaccess — it starts with a dot, no other extension
  4. Click Create New File
  5. Find the new .htaccess file → right-click it → Edit
  6. Delete anything already in the editor, then paste ALL of the text below:
# Hide directory listings from visitors Options -Indexes DirectoryIndex index.html # Force HTTPS — redirects http:// to https:// automatically RewriteEngine On RewriteCond %{HTTPS} off RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301] # Speed — cache images and scripts in visitors' browsers <IfModule mod_expires.c> ExpiresActive On ExpiresByType text/css "access plus 1 month" ExpiresByType application/javascript "access plus 1 month" ExpiresByType image/jpeg "access plus 6 months" ExpiresByType image/png "access plus 6 months" ExpiresByType image/webp "access plus 6 months" ExpiresByType image/svg+xml "access plus 6 months" </IfModule>
  1. Click Save Changes → close the editor

If Something Doesn't Look Right

🖥️ Site shows a blank page or the old WordPress site
Clear your browser cache: press Ctrl + Shift + Delete → clear everything → refresh the page. Also check that index.html is directly inside public_html/, not in a sub-folder.
🔗 City pages give "404 Not Found"
The .htaccess file is missing or saved incorrectly. Go back and redo Step 8 exactly as written.
🎨 Site looks unstyled — no colors, no layout
The styles.css file is in the wrong location. In File Manager, verify that styles.css is directly inside public_html/ — same level as index.html.
📬 Contact form doesn't send leads to GHL
Re-check that you saved the correct GHL webhook URL inside script.js (Step 2). The URL goes between the single quotes on the GHL_WEBHOOK_URL line.
🔙 I want my old WordPress site back
Option 1: Re-install WordPress in GoDaddy (My Products → WordPress → Install), then use UpdraftPlus to restore your backup.

Option 2: Your backup files are in the WP-Backup-OLD folder on your Desktop. Contact GoDaddy support and they can help you restore from it.
